# cosmosx-ai

## Overview
cosmosx-ai provides tooling for collecting, cleaning, enriching, and packaging research articles sourced from NASA and PubMed Central (PMC). The utilities in this repository crawl article pages, normalize and augment their metadata, generate AI-assisted summaries, extract research gap statements, and export the enriched corpus into SQL scripts or retrieval-ready formats.

## Repository contents

### Datasets & intermediate artifacts
- `SB_publication_PMC.csv` and similar CSV files: seed lists of PMC article URLs consumed by the crawlers.
- `info.csv`: consolidated metadata generated by the crawlers; it is repeatedly normalized and deduplicated throughout the workflow.
- `info.parquet`: parquet version of the cleaned metadata after figure filtering and keyword merging; treated as the canonical dataset for downstream steps.
- `data.parquet`: curated metadata consumed by `create_dummy_sql.py` to create SQL insert statements.
- `summary_results/`: runtime directory populated with LLM-generated paper summaries, keyed by PMCID.
- `research_gaps_extracted.csv` / `.parquet`: structured gap analyses parsed from markdown reports for later SQL export.
- `research_id_pmc.csv`: mapping between PMC identifiers and database identifiers, used when producing research-gap SQL statements.

### Core Python utilities
- `pmc_crawler.py`: downloads PMC articles listed in a CSV, extracts metadata and abstracts, writes markdown summaries, and prunes orphaned files.
- `new_pmc_crawler.py`: feature-rich crawler compatible with the original CLI that additionally captures figure URLs, captions, JSON payloads, and sanitized metadata.
- `clean_crawl_data.py`: normalizes figure data, synchronizes markdown availability, merges keyword batches, and emits cleaned CSV/parquet metadata.
- `extract_keyword.py`: applies KeyBERT with SciBERT embeddings and custom stop words to append top keywords for each abstract in the metadata file.
- `auto_gpt.py`: automates ChatGPT interactions via `pyautogui` to submit prompts, capture responses, and persist AI-generated summaries per paper.
- `check_location.py`: helper script streaming live cursor coordinates to calibrate automation hotspots used by `auto_gpt.py`.
- `check_right_json_output.py`: validates markdown exports containing JSON summaries, strips code fences, enforces required keys, and saves corrected `.json` files while logging mismatches.
- `catch_missmatch.py`: reconciles PMC identifiers across parquet metadata, JSON summaries, and markdown folders, deleting stragglers and rewriting the harmonized parquet file.
- `clean_research_gap_data.py`: parses research-gap markdown files with regular expressions to extract conceptual, methodological, and empirical gap statements and exports a tidy CSV table.
- `create_dummy_sql.py`: converts curated parquet metadata into SQLite insert statements for `research` and `figure` tables, normalizing dates and parsing JSON figure data.
- `create_research_gap_dummy.py`: joins gap analyses with research IDs, enforces one-to-many relationships, escapes strings, and produces SQL inserts for the `research_gaps` table with reporting on matches.
- `paper_inspect.py`: loads a PubMedBERT classifier to score sliding-window paragraph chunks, selects high-scoring sections per rhetorical label, and compiles markdown reports.
- `paper_rag.py`: implements a dense retrieval workflow using SPECTER2 adapters, optional cross-encoder reranking, cached paper text, and a CLI-style example search session.
- `main.py`: placeholder entry point that currently prints a greeting.

### SQL scripts
- `populate.sql`: SQLite script that seeds `research` and `figure` tables with the enriched paper content within a single transaction.
- `insert_research_gaps.sql`: SQLite script that batches conceptual, methodological, and empirical research-gap statements linked to research IDs.

### Project configuration
- `pyproject.toml`: project metadata, dependencies, and packaging configuration referencing this README.

## Suggested workflow
1. **Crawl source articles** using `pmc_crawler.py` or `new_pmc_crawler.py`, providing a CSV of PMC URLs to produce markdown files and `info.csv`.
2. **Clean and enrich metadata** with `clean_crawl_data.py` (figure normalization and keyword merging) and `extract_keyword.py` (keyword inference).
3. **Generate and validate AI summaries** by running `auto_gpt.py`, then verify outputs with `check_right_json_output.py` and reconcile identifiers via `catch_missmatch.py`.
4. **Analyze research gaps** using `clean_research_gap_data.py`, followed by SQL export through `create_research_gap_dummy.py`.
5. **Package datasets** for databases with `create_dummy_sql.py`, `populate.sql`, and `insert_research_gaps.sql`, or build retrieval tooling with `paper_rag.py`.
